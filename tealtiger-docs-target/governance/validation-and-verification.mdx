# Governance Validation & Verification (v1.2.x)

TealTiger treats **verification as a first‑class requirement**.

Governance is only real if it can be **proven**.

---

## What Must Be Verified

TealTiger v1.2.x requires validation of:

- Policy compilation
- Runtime decisions
- Evidence artifacts
- Cross‑language parity
- Semantic stability

---

## Golden Corpora

Golden corpora define **contract truth** for:

- Policy compiler output
- Decision outcomes
- EvidenceBundle structure
- LineageGraph ordering

Any deviation is a regression.

---

## CI Gating

CI MUST fail if:

- Golden outputs change unexpectedly
- Reason codes appear without registry entries
- Canonicalization rules are violated

Golden corpora are immutable within a version line.

---

## Cross‑Language Parity

- TypeScript and Python MUST produce identical outputs
- Hashes, ordering, and fields must match exactly

Parity is non‑negotiable.

---

## What This Replaces

Traditional governance relies on:

- Manual reviews
- Screenshots
- Post‑hoc log analysis

TealTiger replaces this with **deterministic verification**.

---

## Summary

Validation is not optional.

In TealTiger, **correctness is enforced by tests**, not trust.
