## Purpose

This file is a **starter template** for creating a **Golden Corpus**.

A Golden Corpus is the canonical source of truth for how policies are expected to behave. It captures **inputs and expected outcomes** so that governance decisions remain:

- Deterministic
- Reproducible
- Auditable
- Safe to evolve

This file is intentionally **minimal and explicit**.

---

## How to Use This Template

1. Copy this file and remove the `Template` status
2. Fill in the test cases relevant to your policies
3. Commit it alongside policy changes
4. Treat it as a **contract**, not an example

---

Policy Scope:

- Security governance
- Reliability governance

Applies To:
- Environments: prod, staging
- Policy versions: v1.1.0
- Enforcement mode: shadow / warn / enforce

Owner:
- Team or service responsible for this corpus

---

## Global Assumptions

- All timestamps are omitted unless explicitly tested
- All inputs are evaluated against the same policy version
- Missing required inputs must result in deterministic safe behavior
- No randomness or time-based logic is permitted

## Rules for Maintaining This Corpus

- ✅ Update the corpus when behavior changes **intentionally**
- ❌ Do not update the corpus to fix failing tests
- ✅ Add new cases for new policies or thresholds
- ❌ Do not remove cases without explicit review

If a policy change breaks the golden corpus, the change is **not safe**.

---

## Review Checklist

- [ ] All golden corpus cases pass
- [ ] New behavior has new test cases
- [ ] Reason codes are explicit and stable
- [ ] Boundary conditions are covered
- [ ] Missing-input behavior is deterministic

## Summary

This golden corpus template is the **foundation of deterministic governance**.

If policies define *what should happen*,
