# Docker Containers

TealTiger provides official Docker images for both Python and TypeScript SDKs, available on both Docker Hub and GitHub Container Registry (GHCR). Use containers for consistent deployments, CI/CD pipelines, development environments, and serverless functions.

## Why use Docker?

- **Consistent environments** - Same runtime across dev, staging, and production
- **Fast onboarding** - Get started in 5 minutes instead of 2 hours
- **CI/CD ready** - Drop-in containers for GitHub Actions, GitLab CI, Jenkins
- **Serverless compatible** - Deploy to AWS Lambda, Google Cloud Run, Azure Functions
- **Multi-platform** - Supports both amd64 and arm64 architectures

---

## Quick Start

<CodeGroup>
```bash Python SDK
# Pull the latest image (Docker Hub)
docker pull tealtigeradmin/tealtiger-python:latest

# Run interactively
docker run -it --rm \
  -e OPENAI_API_KEY=your-key \
  tealtigeradmin/tealtiger-python:latest \
  python
```

```bash TypeScript SDK
# Pull the latest image (Docker Hub)
docker pull tealtigeradmin/tealtiger-typescript:latest

# Run interactively
docker run -it --rm \
  -e OPENAI_API_KEY=your-key \
  tealtigeradmin/tealtiger-typescript:latest \
  node
```
</CodeGroup>

## Image Variants Explained

### Production

Optimized for production deployments with minimal size and maximum security.

**Features:**
- Multi-stage build for smaller image size
- Non-root user execution
- Production dependencies only
- No development tools

**Best for:**
- Production deployments
- Serverless functions (Lambda, Cloud Run)
- Container orchestration (Kubernetes, ECS)
- CI/CD test runners

### Development

Includes development tools for local development and testing.

**Features:**
- All production features
- Development tools (pytest, mypy, black, eslint, typescript)
- Git and vim for editing
- VS Code dev container support

**Best for:**
- Local development
- VS Code dev containers
- Testing and debugging
- CI/CD with linting/type checking

### Alpine (Python only)

Minimal production image based on Alpine Linux.

**Features:**
- Smallest image size
- Alpine Linux base
- Full SDK functionality
- Production-ready

**Best for:**
- Size-constrained deployments
- Edge computing
- IoT devices
- Cost optimization (faster pulls)

### Jupyter (Python only)

Interactive development environment with Jupyter Lab.

**Features:**
- Jupyter Lab pre-installed
- Data science tools (pandas, numpy)
- Port 8888 exposed
- Notebook examples included

**Best for:**
- Interactive experimentation
- Data analysis
- Prototyping
- Training and demos

## Usage Examples

### Run Your Script

Mount your code and run it in a container.

<CodeGroup>
```bash Python
docker run --rm \
  -v $(pwd):/workspace \
  -w /workspace \
  -e OPENAI_API_KEY=sk-xxx \
  tealtigeradmin/tealtiger-python:latest \
  python my_script.py
```

```bash TypeScript
docker run --rm \
  -v $(pwd):/workspace \
  -w /workspace \
  -e OPENAI_API_KEY=sk-xxx \
  tealtigeradmin/tealtiger-typescript:latest \
  node my_script.js
```
</CodeGroup>

### Interactive Shell

Start an interactive Python or Node.js shell.

<CodeGroup>
```bash Python
docker run -it --rm \
  -e OPENAI_API_KEY=sk-xxx \
  tealtigeradmin/tealtiger-python:latest \
  python

# In Python:
>>> from tealtiger import TealOpenAI
>>> client = TealOpenAI(api_key="sk-xxx")
```

```bash TypeScript
docker run -it --rm \
  -e OPENAI_API_KEY=sk-xxx \
  tealtigeradmin/tealtiger-typescript:latest \
  node

// In Node.js:
> const { TealOpenAI } = require('tealtiger');
> const client = new TealOpenAI({ apiKey: 'sk-xxx' });
```
</CodeGroup>

### Jupyter Notebook

Start Jupyter Lab for interactive development.

```bash
docker run -p 8888:8888 \
  -e OPENAI_API_KEY=sk-xxx \
  -v $(pwd)/notebooks:/app/notebooks \
  tealtigeradmin/tealtiger-python:jupyter

# Open browser to http://localhost:8888
```

### Development Environment

Start a development container with all tools.

<CodeGroup>
```bash Python
docker run -it --rm \
  -v $(pwd):/app \
  -e OPENAI_API_KEY=sk-xxx \
  tealtigeradmin/tealtiger-python:dev \
  bash

# Inside container:
$ pytest tests/
$ black src/
$ mypy src/
```

```bash TypeScript
docker run -it --rm \
  -v $(pwd):/app \
  -e OPENAI_API_KEY=sk-xxx \
  tealtigeradmin/tealtiger-typescript:dev \
  bash

# Inside container:
$ npm test
$ npm run lint
$ npm run build
```
</CodeGroup>

## Docker Compose

Create a `docker-compose.yml` file for easier management.

<CodeGroup>
```yaml Python
version: '3.8'

services:
  tealtiger:
    image: tealtigeradmin/tealtiger-python:latest
    environment:
      - OPENAI_API_KEY=${OPENAI_API_KEY}
      - ANTHROPIC_API_KEY=${ANTHROPIC_API_KEY}
    volumes:
      - ./workspace:/workspace
    working_dir: /workspace
    command: python my_script.py
```

```yaml TypeScript
version: '3.8'

services:
  tealtiger:
    image: tealtigeradmin/tealtiger-typescript:latest
    environment:
      - OPENAI_API_KEY=${OPENAI_API_KEY}
      - ANTHROPIC_API_KEY=${ANTHROPIC_API_KEY}
    volumes:
      - ./workspace:/workspace
    working_dir: /workspace
    command: node my_script.js
```
</CodeGroup>

Run with:
```bash
docker-compose run tealtiger
```

## CI/CD Integration

### GitHub Actions

<CodeGroup>
```yaml Python
name: Test with TealTiger

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    container:
      image: tealtigeradmin/tealtiger-python:1.1.1
    
    steps:
      - uses: actions/checkout@v3
      
      - name: Run tests
        env:
          OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}
        run: python -m pytest tests/
```

```yaml TypeScript
name: Test with TealTiger

on: [push, pull_request]

    container:
```

## Security Best Practices

### Non-Root User

All images run as non-root user for security.

<CodeGroup>
```bash Python
# Verify non-root user
docker run --rm tealtigeradmin/tealtiger-python:latest whoami
# Output: tealtiger
```

```bash TypeScript
# Verify non-root user
docker run --rm tealtigeradmin/tealtiger-typescript:latest whoami
# Output: node
```
</CodeGroup>

### Resource Limits

Limit CPU and memory usage.

```bash
docker run --rm \
  --cpus="0.5" \
  --memory="512m" \
  -e OPENAI_API_KEY=sk-xxx \
  tealtigeradmin/tealtiger-python:latest \
  python my_script.py
```

### Read-Only Filesystem

Run with read-only filesystem for extra security.

```bash
docker run --rm \
  --read-only \
  --tmpfs /tmp:size=100M \
  -e OPENAI_API_KEY=sk-xxx \
  tealtigeradmin/tealtiger-python:latest \
  python my_script.py
```

### Secrets Management

Never hardcode secrets in images. Use environment variables or secret managers.

```bash
# Good: Use environment variables
docker run --rm \
  -e OPENAI_API_KEY=$(cat ~/.openai/key) \
  tealtigeradmin/tealtiger-python:latest \
  python my_script.py

# Better: Use Docker secrets (Swarm/Kubernetes)
docker service create \
  --secret openai_key \
  tealtigeradmin/tealtiger-python:latest
```

## VS Code Dev Containers

Use TealTiger images with VS Code dev containers for consistent development environments.

### Setup

1. Install the **Dev Containers** extension in VS Code
2. Create `.devcontainer/devcontainer.json`:

<CodeGroup>
```json Python
{
  "name": "TealTiger Python Dev",
  "image": "tealtigeradmin/tealtiger-python:dev",
  
  "customizations": {
    "vscode": {
      "extensions": [
        "ms-python.python",
        "ms-python.vscode-pylance",
        "ms-python.black-formatter"
      ],
      "settings": {
        "python.linting.enabled": true,
        "python.linting.pylintEnabled": true,
        "python.formatting.provider": "black"
      }
    }
  },
  
  "forwardPorts": [8888],
  "postCreateCommand": "pip install -e .",
  "remoteUser": "tealtiger"
}
```

```json TypeScript
{
  "name": "TealTiger TypeScript Dev",
  "image": "tealtigeradmin/tealtiger-typescript:dev",
  
  "customizations": {
    "vscode": {
      "extensions": [
        "dbaeumer.vscode-eslint",
        "esbenp.prettier-vscode"
      ],
      "settings": {
        "editor.formatOnSave": true,
        "editor.defaultFormatter": "esbenp.prettier-vscode"
      }
    }
  },
  
  "forwardPorts": [3000],
  "postCreateCommand": "npm install",
  "remoteUser": "node"
}
```
</CodeGroup>

3. Open Command Palette (Ctrl+Shift+P)
4. Select **Dev Containers: Reopen in Container**

## Building Locally

Build images locally for customization or testing.

<CodeGroup>
```bash Python
# Clone the repository
git clone https://github.com/agentguard-ai/tealtiger-python.git
cd tealtiger-python

docker build -t tealtigeradmin/tealtiger-python:latest .
```

```bash TypeScript
cd tealtiger-typescript

# Build production image
docker build -t tealtigeradmin/tealtiger-typescript:latest .

# Build dev image
docker build -t tealtigeradmin/tealtiger-typescript:dev -f Dockerfile.dev .
```

## Troubleshooting

### Image Pull Fails

If Docker Hub is unavailable, try the GHCR mirror:

```bash
# Python
docker pull ghcr.io/agentguard-ai/tealtiger-python:latest

# TypeScript
docker pull ghcr.io/agentguard-ai/tealtiger-typescript:latest
```

### Permission Denied

Run as your current user:

```bash
docker run --rm \
  --user $(id -u):$(id -g) \
  -v $(pwd):/workspace \
  tealtigeradmin/tealtiger-python:latest \
  python my_script.py
```

### Import Errors

Verify the SDK is installed:

<CodeGroup>
```bash Python
docker run --rm tealtigeradmin/tealtiger-python:latest \
  python -c "import tealtiger; print(tealtiger.__version__)"
```

```bash TypeScript
docker run --rm tealtigeradmin/tealtiger-typescript:latest \
  node -e "console.log(require('tealtiger'))"
```
</CodeGroup>

### Slow Builds

Use Docker BuildKit for faster builds:

```bash
export DOCKER_BUILDKIT=1
docker build -t my-image .
```

### Build Times

- **Cold cache**: 2-10 minutes

- **Container start**: <2 seconds

### Pull Times

- **Fast connection (100 Mbps)**: 10-30 seconds
- **Average connection (25 Mbps)**: 30-90 seconds
- **Slow connection (5 Mbps)**: 2-5 minutes
