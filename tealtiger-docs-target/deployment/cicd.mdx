# CI/CD Integration

Use TealTiger Docker containers and official integrations in your CI/CD pipelines for consistent testing and deployment.

---

## GitHub Actions

### Official TealTiger Action

The recommended way to add TealTiger security scanning to GitHub Actions:

```yaml
name: TealTiger Security Scan

on: [push, pull_request]

jobs:
  security-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Run TealTiger Scan
        uses: agentguard-ai/tealtiger-action@v1
        with:
          scan-path: ./src
          guardrails: pii,prompt-injection,content-moderation
          sensitivity: medium
          policy-file: ./tealtiger-policy.yml
          fail-on-finding: true
          report-format: sarif
```

**Inputs:**

| Input | Description | Default |
|-------|-------------|---------|
| `scan-path` | Directory or file glob to scan | (required) |
| `guardrails` | Comma-separated guardrails to enable | `pii,prompt-injection,content-moderation` |
| `sensitivity` | Scan sensitivity: low, medium, high | `medium` |
| `policy-file` | Path to policy configuration file | `""` |
| `fail-on-finding` | Fail workflow on violation | `true` |
| `report-format` | Output format: json, junit, sarif | `json` |

### Container-Based Workflows

You can also use TealTiger images directly as job containers:

<CodeGroup>
```yaml Python SDK
name: Test with TealTiger

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    container:
      image: tealtigeradmin/tealtiger-python:1.1.1
    
    steps:
      - uses: actions/checkout@v3
      
      - name: Install dependencies
        run: pip install -r requirements.txt
      
      - name: Run tests
        env:
          OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}
        run: python -m pytest tests/
      
      - name: Run policy tests
        run: python -m tealtiger test policies/
```

```yaml TypeScript SDK
name: Test with TealTiger

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    container:
      image: tealtigeradmin/tealtiger-typescript:1.1.1
    
    steps:
      - uses: actions/checkout@v3
      
      - name: Install dependencies
        run: npm ci
      
      - name: Run tests
        env:
          OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}
        run: npm test
      
      - name: Run policy tests
        run: npx tealtiger test policies/
```
</CodeGroup>

## GitLab CI

### Official Reusable Template

Include the official TealTiger GitLab CI template for turnkey security scanning:

```yaml
include:
  - remote: 'https://raw.githubusercontent.com/agentguard-ai/tealtiger/main/integrations/cicd/gitlab-ci/.gitlab-ci.yml'

variables:
  TEALTIGER_SCAN_PATH: "./src"
  TEALTIGER_GUARDRAILS: "pii,prompt-injection,content-moderation"
  TEALTIGER_SENSITIVITY: "medium"
  TEALTIGER_FAIL_ON_FINDING: "true"
```

This template provides two jobs:
- **tealtiger-scan** — Runs guardrail scanning with JUnit and Code Quality reports
- **tealtiger-policy-test** — Runs policy tests (when `TEALTIGER_POLICY_FILE` is set)

Both jobs use the `tealtigeradmin/tealtiger-docker:latest` image.

## CircleCI

### Official TealTiger Orb

Use the TealTiger CircleCI orb for integrated security scanning:

```yaml
version: 2.1

orbs:
  tealtiger: tealtiger/security-scan@1.0

workflows:
  security:
    jobs:
      - tealtiger/full-scan:
          scan-path: ./src
          guardrails: pii,prompt-injection,content-moderation
          sensitivity: medium
          fail-on-finding: true
```

The orb uses the `tealtigeradmin/tealtiger-docker:latest` executor image and provides:
- **scan** command — Run guardrail scans
- **policy-test** command — Run policy tests
- **full-scan** job — Combined guardrail + policy scan

## Best Practices

### Use Specific Tags

Don't use `latest` in production CI/CD. Pin to specific versions:

```yaml
container:
  image: tealtigeradmin/tealtiger-python:1.1.1
```

### Cache Dependencies

Speed up builds by caching dependencies:

```yaml
# GitHub Actions
- uses: actions/cache@v3
  with:
    path: ~/.cache/pip
    key: ${{ runner.os }}-pip-${{ hashFiles('requirements.txt') }}
```

### Parallel Testing

Run tests in parallel for faster feedback:

```yaml
# GitHub Actions
strategy:
  matrix:
    test-group: [unit, integration, policy]
```

### Security Scanning

Scan containers for vulnerabilities:

```yaml
# GitHub Actions
- name: Run Trivy scanner
  uses: aquasecurity/trivy-action@master
  with:
    image-ref: 'tealtigeradmin/tealtiger-python:1.1.1'
    format: 'sarif'
    output: 'trivy-results.sarif'
```
