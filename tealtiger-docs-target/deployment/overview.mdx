# Deployment Overview

TealTiger provides flexible deployment options for every use case, from serverless functions to enterprise Kubernetes clusters. Choose the deployment pattern that fits your architecture.

## Deployment Philosophy

TealTiger follows a **zero-infrastructure** philosophy:
- ✅ No servers to manage
- ✅ No databases to maintain
- ✅ No complex setup required
- ✅ Drop-in SDK integration

However, we provide comprehensive deployment tooling for teams that need:
- Container orchestration
- Multi-region deployments
- Enterprise-grade monitoring
- Cost optimization
- Compliance and audit trails

## Deployment Patterns

### 1. Serverless Functions

Deploy TealTiger as serverless functions for event-driven architectures.

**Supported Platforms:**
- AWS Lambda (Container images + Lambda layers)
- Google Cloud Run (Fully managed containers)
- Azure Functions (Custom containers)
- Vercel Edge Functions
- Cloudflare Workers

**Use Cases:**
- API endpoints
- Webhook handlers
- Scheduled jobs
- Event processing
- Chatbot backends

**Pros:**
- Zero infrastructure management
- Auto-scaling
- Pay-per-use pricing
- Fast cold starts (<2s)

**Cons:**
- Execution time limits
- Memory constraints
- Cold start latency

[Learn more →](/deployment/serverless)

Deploy TealTiger across multiple cloud providers for redundancy and cost optimization.

Deploy TealTiger to multiple regions for:

## Lambda Layers

Pre-built Lambda layers for zero-dependency deployments.

### Available Layers

TealTiger provides Lambda layers for all AWS regions:

**Python Runtimes:**
- Python 3.9, 3.10, 3.11, 3.12, 3.13

**Node.js Runtimes:**
- Node.js 18.x, 20.x, 22.x

**Regions:**
- All 33 AWS commercial regions
- GovCloud regions (coming soon)

### Layer Registry

Browse and download layers at: **[layers.tealtiger.ai](https://layers.tealtiger.ai)**

```bash
# Get layer ARN for Python 3.11 in us-east-1
curl https://layers.tealtiger.ai/api/layers/python3.11/us-east-1

# Response:
{
  "runtime": "python3.11",
  "region": "us-east-1",
  "version": "1.1.1",
  "arn": "arn:aws:lambda:us-east-1:123456789012:layer:tealtiger-python311:1"
}
```

## Cost Optimization

Optimize TealTiger deployment costs across all platforms.

### Cost Tracking

TealTiger provides built-in cost tracking:

```typescript
import { TealMonitor } from 'tealtiger';

const monitor = new TealMonitor({
  costTracking: {
    enabled: true,
    budget: {
      monthly: 1000, // $1000/month
      alerts: ['email@example.com']
    }
  }
});

// Track costs per request
const cost = await monitor.getCost({
  provider: 'openai',
  model: 'gpt-4',
  tokens: 1500
});

console.log(`Request cost: $${cost.toFixed(4)}`);
```

### Cost Optimization Strategies

1. **Provider Selection**: Route to cheapest provider
2. **Model Selection**: Use smaller models when possible
3. **Caching**: Cache responses to reduce API calls
4. **Batching**: Batch requests for better pricing
5. **Region Selection**: Use cheaper regions

[Learn more →](/deployment/cost-optimization)

## Performance Optimization

Optimize TealTiger performance for your workload.

### Performance Tuning

1. **Cold Start Optimization**
   - Use provisioned concurrency (Lambda)
   - Keep containers warm (Cloud Run)
   - Use smaller images (Alpine variants)

2. **Memory Optimization**
   - Right-size memory allocation
   - Use memory profiling
   - Enable garbage collection tuning

3. **Network Optimization**
   - Use regional endpoints
   - Enable HTTP/2
   - Implement connection pooling

4. **Caching**
   - Response caching
   - Model caching
   - Configuration caching

[Learn more →](/deployment/performance)
