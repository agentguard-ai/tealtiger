# TealTiger v1.1.1

TealTiger v1.1.1 is a patch release that delivers enterprise-grade AI agent security through a zero-infrastructure SDK. This release builds on v1.1.0 with full multi-provider parity, enhanced stability, and production hardening.

---

## What's New in v1.1.1

### Multi-Provider Feature Parity

Both TypeScript and Python SDKs now support all 7 providers with identical capabilities:

| Provider | Client | Status |
|----------|--------|--------|
| OpenAI | `TealOpenAI` | ✅ Full parity |
| Anthropic | `TealAnthropic` | ✅ Full parity |
| Google | `TealGemini` | ✅ Full parity |
| AWS Bedrock | `TealBedrock` | ✅ Full parity |
| Azure OpenAI | `TealAzureOpenAI` | ✅ Full parity |
| Mistral AI | `TealMistral` | ✅ Full parity |
| Cohere | `TealCohere` | ✅ Full parity |

### Enterprise Adoption Features

- **Policy Rollout Modes** — REPORT_ONLY → MONITOR → ENFORCE gradual deployment
- **Deterministic Decision Contract** — Typed Decision object with action, reason_codes, risk_score, correlation_id
- **Correlation IDs & Traceability** — UUID v4 auto-generation, OpenTelemetry-compatible, HTTP header propagation
- **Audit Redaction Guarantees** — 5 redaction levels (FULL, CATEGORY_ONLY, SIZE_ONLY, HASH, NONE), HASH by default
- **Policy Test Harness** — CI/CD-ready test runner with JUnit XML export and built-in test corpora
- **Cost Governance** — Per-request/session/agent/tenant budgets with spend velocity anomaly detection

### OWASP Coverage

7 out of 10 OWASP Top 10 for Agentic Applications vulnerabilities covered through SDK-only architecture.

## Migration from v1.1.0

v1.1.1 is a drop-in replacement for v1.1.0. No code changes required.

```bash
# TypeScript
npm install tealtiger@latest

# Python
pip install --upgrade tealtiger
```

All v1.1.0 APIs, schemas, and behaviors are preserved. New features are opt-in.

## Stability Guarantees

All v1.1.0 stability guarantees apply to v1.1.1:

- ✅ Deterministic decisions — same input + same policy = same output
- ✅ Stable audit schema — `tealtiger.audit_event@1.1.0` unchanged
- ✅ Immutable IDs — `stable_id` values never change meaning
- ✅ Backward compatibility — v1.1.1 works with v1.1.0 policies
- ✅ Additive changes only — new features, no breaking changes

See [Stability Guarantees](/versioning/stability-guarantees) for details.
