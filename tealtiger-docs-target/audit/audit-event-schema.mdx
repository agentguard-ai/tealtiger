## Purpose

TealTiger v1.1.0 includes **audit and telemetry** to make policy decisions:

- **Transparent** (what happened)
- **Explainable** (why it happened)
- **Traceable** (who/what initiated it)
- **Auditable** (reproducible evidence)

In v1.1.0, telemetry focuses on **policy decision events** (allow/restrict/block) and **action execution events** (redaction/limits/routing/logging).

This page defines the **Audit Event Schema** used by the SDK/runtime to emit structured audit records.

Audit and telemetry in v1.1.0 are designed to:

- Record **policy evaluation inputs and outputs** (without leaking secrets)
- Record **actions executed** (and their outcomes)
- Provide **reason codes** and **risk scores** for explainability
- Enable correlation across distributed systems via **trace IDs**
- Support export to log stores / SIEM / data lakes

---

> **Upstream:** Decisions originate in /concepts/decision-lifecycle and are defined by policies in /policy/policy-authoring-guide.

## Schema Overview

```yaml
machine_spec:
  version: v1.1.0
  id: tealtiger.audit_event.schema
  type: audit_event_schema
  schema:
    name: tealtiger.audit_event
    version: "1.1.0"
  event_types:
    - policy.decision
    - policy.action
    - policy.error
    - policy.metric
  required_top_level_fields:
    - schema
    - event
    - correlation
    - actor
    - execution
    - policy
    - reason_codes
    - actions
    - redaction
    - extensions
  stability:
    promise: stable_within_v1_1_x
    breaking_changes: forbidden
    extensibility:
      allowed_namespace: extensions
      rule: additions_only
```

### Schema identity

Each audit event includes schema identifiers for compatibility:

- `schema.name`: fixed logical name for this schema
- `schema.version`: semantic version of the event schema (v1.1.0 here)
- `event.type`: event category (decision/action/error/metric)

## Event Structure

```yaml
machine_spec:
  version: v1.1.0
  id: tealtiger.audit_event.required_fields
  type: required_fields
  schema_fields:
    schema.name: string
    schema.version: string
    event.id: string
    event.type: string
    event.timestamp: string
    event.severity: string
    event.outcome: string
    correlation.trace_id: string
    correlation.span_id: string
    policy.policy_id: string
    policy.policy_version: string
    policy.mode: string
    policy.decision: string
```

> This is the **canonical event shape** emitted by the SDK/runtime.  
> Your implementation may add fields under `extensions.*` as needed, without breaking compatibility.
