# Visual Decision Mapping

## One visual mental model

```mermaid
flowchart TD
  A["Request / Prompt / Tool intent"] --> B["Collect Context (identity, env, tool metadata)"]
  B --> C["Cost checks (budgets, token caps)"]
  C --> D["Security checks (PII, tool access, exfil patterns)"]
  D --> E["Reliability checks (timeouts, retries, fallbacks)"]
  E --> F{"Decision.action"}
  F -->|ALLOW| G["Proceed"]
  F -->|DENY| H["Block (or record only in MONITOR)"]
  F -->|REDACT| I["Redact then proceed"]
  F -->|TRANSFORM| J["Transform then proceed"]
  F -->|DEGRADE| K["Degrade then proceed"]
  F -->|REQUIRE_APPROVAL| L["Defer to approval workflow"]
  F --> M["Emit AuditEvent (evidence)"]
```

<Callout type="info">
This diagram is intentionally high-level. The stable contract is the Decision Model: action + reason codes + optional risk.
</Callout>

---

## How to interpret the diagram

- Governance checks are **deterministic** and policy-driven.
- The Decision object is the **single application-facing outcome**.
- Reason codes and risk scores explain *why*.
- Audit events record evidence for later review.
