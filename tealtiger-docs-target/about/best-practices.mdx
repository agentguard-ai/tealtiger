## Core Best Practices

### 1. Treat Policies as Governance Contracts

Policies should be:
- Explicit
- Reviewed
- Versioned
- Stable over time

**Best practice:**
- Treat policies like API contracts
- Require review before changes
- Avoid frequent, ad‑hoc edits

Policies define **guardrails**, not experimentation.

---

### 2. Keep Policies Deterministic and Explainable

**Best practice:**
- Use clear conditions
- Avoid ambiguous thresholds
- Ensure every decision has a clear reason

Every policy decision should answer:
> “Why did this pass or fail?”

If the answer is unclear, the policy is too complex.

---

### 3. Pass Context Through Inputs, Not Policy Logic

**Best practice:**
- Encode dynamic behavior in inputs (metadata, execution context)
- Keep policy logic stable

Examples of good inputs:
- Environment (prod, staging)
- Execution identity
- Cost metadata
- Request intent

Policies should interpret context — not generate it.

---

### 4. Separate Policies by Concern

**Best practice:**
- Cost policies → cost constraints only
- Security policies → risk and exposure
- Reliability policies → stability and limits

Avoid monolithic policies.

Composable policies are:
- Easier to test
- Easier to audit
- Safer to evolve

---

### 5. Use Audit Events as a First‑Class Output

**Best practice:**
- Treat audit events as durable records
- Forward them to SIEM / logging systems
- Preserve structure and meaning

Audit logs are **not debug logs** — they are governance evidence.

---

### 6. Integrate TealTiger Early in the Execution Path

**Best practice:**
- Evaluate policies before AI actions execute
- Fail fast when constraints are violated

Early enforcement:
- Reduces cost
- Reduces blast radius
- Improves predictability

---

### 7. Keep Policies Vendor‑Neutral

**Best practice:**
- Avoid provider‑specific logic
- Base decisions on behavior and risk
- Design policies that survive vendor changes

This ensures:
- Portability
- Long‑term policy reuse
- Infrastructure flexibility

---

### 8. Version Policies Alongside Code

**Best practice:**
- Store policies in source control
- Tag policies with versions
- Release policies alongside application changes

This preserves:
- Traceability
- Reproducibility
- Audit confidence

---

### 9. Start Simple, Then Expand Coverage

**Best practice:**
- Begin with a small number of high‑impact policies
- Observe behavior
- Expand deliberately

Over‑governing early leads to friction and distrust.

---

### 10. Review Policies as Systems Evolve

AI systems change.
Usage patterns change.
Risk changes.

**Best practice:**
- Periodically review policies
- Validate assumptions
- Adjust with intent

Governance is a **continuous discipline**, not a one‑time setup.

---

## Mental Model to Keep

TealTiger works best when it is:

- Predictable
- Boring
- Explicit
- Composable

If governance feels magical or surprising, something is wrong.
