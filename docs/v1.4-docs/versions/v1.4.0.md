# v1.4.0 — Zero-Config Observe Mode

**Release Date:** 2026-06-15

TealTiger v1.4 is the largest release since v1.1. The headline feature — `observe()` — eliminates the adoption barrier entirely: wrap your LLM client in one line, get full-stack visibility immediately, configure nothing.

---

## New Features

### 🔭 observe() — Zero-Config Governance Entry Point

A single function that wraps any supported LLM provider client with cost tracking, audit logging, behavioral baseline construction, and PII detection. No configuration files, no policy definitions, no external services.

```typescript
import { observe } from "tealtiger";
const client = observe(new OpenAI());
```

- Supports all 14 providers (OpenAI, Anthropic, Gemini, Vertex AI, Bedrock, Azure OpenAI, Cohere, Mistral, DeepSeek, Groq, xAI, Together, HF-TGI, and more)
- Automatic cost tracking per request, session, and agent
- Behavioral baseline construction from first 100 requests
- PII detection in REPORT_ONLY mode — never blocks, always logs
- Emergency kill switch: `freeze()` / `unfreeze()`

---

### 🛡️ Multi-Stage Defense Pipeline

TealGuard now decomposes defense into three specialized stages with configurable depth:

| Depth | Stages | Target Latency |
|-------|--------|----------------|
| `fast` | Stage 1 only (pattern scanning) | < 5ms |
| `standard` | Stage 1 + Stage 2 (structural analysis) | < 20ms |
| `deep` | Stage 1 + Stage 2 + Stage 3 (local classifier) | < 50ms |

Short-circuits on DENY — if Stage 1 catches it, Stages 2 and 3 never run.

---

### 🔄 Post-Execution Response Governance

Bi-directional scanning: block dangerous inputs before they reach the LLM **and** block sensitive outputs before they reach the caller.

```typescript
const guard = new TealGuard({
  guardrails: {
    pre: { pii: true, secrets: true },
    post: { pii: true, secrets: true, content: true }
  }
});
```

---

### 👤 Role-Based Per-Agent Governance

Define different policies per agent role. Each agent operates with minimum privilege for its function.

```typescript
const policy = perRole({
  researcher: { allowed_tools: ["search", "read"], max_cost: 1.0 },
  writer: { allowed_tools: ["write", "edit"], max_cost: 0.5 },
  reviewer: { allowed_tools: ["read", "comment"], max_cost: 0.2 }
});
```

---

### 📊 Governance Dashboard

A real-time dashboard for visibility into governance decisions, agent status, costs, and audit trails.

- **Overview**: Top-level metrics, active agents, recent decisions
- **Agents**: Role-based agent matrix with freeze controls
- **Costs**: Burn rate, budget forecast, optimization suggestions
- **Policies**: Visual policy editor with simulation mode
- **Audit Trail**: Chronological event log with filtering
- **Evidence Viewer**: Full TEEC envelope inspection
- **Settings**: Budget, velocity, and threshold configuration

## Additional Features

- **Observability Platform Export**: Export governance decisions to Langfuse, AgentOps, Phoenix/OTel
- **RESAMPLE / DEFER_TO_TRUSTED actions**: Recovery beyond binary ALLOW/DENY
- **Canary-Based Honeypot Detection**: Embed canary markers to detect adversarial agents
- **Cost Velocity Alerts**: Detect runaway spend in real-time
- **Budget Exhaustion Forecasting**: Predict when budgets will be exhausted
- **Cost-Aware Model Routing**: Automatically route to cheaper models when cost ceiling is hit
- **AG-UI / A2UI Protocol Governance**: Govern agent-to-frontend event streams
- **Audit Completeness Evidence**: Monotonic sequencing and omission detection
- **Session Sealing**: Terminal seal records for tail-drop detection

---

## Breaking Changes

**None.** v1.4 is fully backward compatible with v1.3. All existing configurations, policies, and integrations continue to work without modification.

- Default `depth` is `fast` (equivalent to v1.3 behavior)
- No `post` guardrails run unless explicitly configured
- `observe()` is additive — does not affect existing `TealGuard` or `TealEngine` usage

## New Dependencies

| Package | Version | Purpose |
|---------|---------|---------|
| `onnxruntime-node` | ^1.17.0 | Stage 3 local binary classifier (deep mode only) |
| `@tealtiger/onnx-model` | ^1.0.0 | Pre-trained harmful content classifier weights |
| N/A (Python) | `onnxruntime >= 1.17` | Python equivalent for Stage 3 |

The ONNX dependency is **optional** — only loaded when `depth: "deep"` is configured. `fast` and `standard` modes have zero new runtime dependencies.
